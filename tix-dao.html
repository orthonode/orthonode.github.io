Overview

What TIX-DAO Solves

Venue ticketing is broken — centralized platforms, rampant scalping, zero provenance, no community voice. TIX-DAO replaces all of it with programmable, verifiable, community-owned infrastructure.

🎫

NFT Ticket Issuance

Each ticket is a Solana NFT minted at purchase. Ownership is on-chain, transferable, and provably scarce. No duplicates possible.

🔐

Hardware Verification

Gate entry checks ticket against Orthonode hardware attestation. A forged or copied ticket fails Gate 1 — hardware identity cannot be spoofed.

🗳️

SPL-Governance

Venue policies, refund rules, and event parameters are set via on-chain proposals. TIX token holders vote. No admin backdoor.

📋

Attendance Records

Every scan is a Solana transaction. Attendance is immutable, auditable, and usable for future governance weight calculations.

Ticket Lifecycle

From Purchase to Entry — Fully On-Chain

🛒

Purchase

Buyer pays SOL. Smart contract mints NFT ticket to buyer wallet. Metadata stored on-chain including event, seat, timestamp.

🔗

Anchor Bind

Ticket NFT is linked to buyer's hardware identity via Orthonode attestation. Binding stored in program account.

📲

Gate Scan

Entry scanner reads NFT + hardware signature. Validates binding on-chain. Pass = hardware + NFT match. Fail = reject.

✅

Attendance TX

Successful entry writes attendance transaction. Immutable record used for governance weight in future proposals.

Architecture

Program Design

01

tix_ticket Program

Core Anchor program managing ticket NFT minting, hardware binding, and entry verification. Stores ticket state PDA per event + buyer combination. Validates attestation receipts from Orthonode SHA layer before confirming entry.

02

tix_governance Program

Wraps SPL-Governance for venue-specific proposals. TIX token holders submit proposals for event parameters, refund policies, and venue rule changes. Attendance weight multiplies base token weight — frequent attendees have more governance voice.

03

tix_treasury Program

Manages ticket revenue, refund pools, and DAO treasury. Revenue split configurable via governance. Emergency refund proposals can unlock treasury within 24-hour timelock.

FAQ

Common Questions

TIX-DAO is a Solana-based SPL-Governance system for venue ticketing. It issues hardware-verified NFT tickets, records attendance on-chain, and enables token-weighted governance over venue policies. Built for Graveyard Hackathon 2026 as a Type B competition project by Orthonode Systems.
Each ticket NFT is anchored to the buyer's hardware identity via Orthonode's SHA attestation layer. Entry verification checks both the NFT ownership and the hardware binding on-chain. A forged or transferred ticket fails Gate 1 because the hardware signature cannot be replicated — it is tied to the specific device at mint time.
TIX-DAO is currently deployed on Solana devnet with a live UI at tix-dao.vercel.app. It is a sandboxed competition project. Mainnet deployment requires explicit authorization from Orthonode Systems leadership and a completed security audit of all Anchor programs.
Venue stakeholders hold TIX governance tokens. Proposals for event parameters, refund policies, and venue rule changes are submitted and voted on via SPL-Governance. Base token weight is multiplied by an attendance factor — frequent attendees carry more governance voice. All decisions are executed on-chain with no admin override path.
